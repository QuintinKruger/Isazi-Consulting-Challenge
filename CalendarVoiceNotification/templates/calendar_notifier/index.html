The Django development server uses the HTTP protocol while the link that automatically sends the user to authrorize the use of the user's calendar uses HTTPS, the browser thus blocks access using this method. The alternative way to get the required token.pickle file for granting access to the user's google calendar is discussed below.

  1. Navigate to localhost:8000/calendar_notifier/get_calendar.
  2. Navigate to the terminal where the the python3 manage.py runserver command was run.
  3. A link will be present in the terminal that must be opened and followed (look for the line that reads Please visit this URL to authorize this application: ...).
  4. The token.pickle file will be saved to the same directory as the manage.py and is then automatically used to give acess to the application.
